国際化の実装
	1. 主にJSTLライブラリを導入することで実現します。
Maven Central Repository（Mavenセントラルリポジトリ）のウェブサイトで「jstl」を検索し、バージョン1.2の.jarファイルをダウンロードします。
ダウンロードした「jstl-1.2.jar」ファイルを、プロジェクト内のsrc/main/webapp/WEB-INF/libフォルダに直接コピー＆ペースト（配置）します。
	2. 言語リソースファイル（.properties）の作成
src/main/javaフォルダ配下に、以下の3つのファイルを新規作成します：「messages.properties」、「messages_zh.properties」、「messages_ja.properties」。
// messagesという名前はカスタム（任意）であり、自由に変更可能です。「_zh」のような部分は固定フォーマットで、対応する言語のファイルを表します。
	3. JSPに追加する必要があるコード
まずライブラリをインポートします。
Java

<%@ taglib prefix="c" uri="http://java.sun.com/jsp/jstl/core" %>
<%@ taglib prefix="fmt" uri="http://java.sun.com/jsp/jstl/fmt" %>

次に、URLでユーザーが言語を指定しているか判断します。
Java

<c:if test="${not empty param.lang}">
    <c:set var="language" value="${param.lang}" scope="session" />
</c:if>

<fmt:setLocale value="${sessionScope.language}" />
<fmt:setBundle basename="messages" />

そして、国際化が必要なテキストを以下の記述に置き換えます。
Java

<fmt:message key="page.title.login" />
<input type="submit" value="<fmt:message key='button.login' />" />
	4. 他のページも国際化対応させる方法
理論上は、ステップ3のコードをもう一度書けばよいのですが、これは非常に面倒です。
JSPの先頭にあるJSTLタグライブラリの宣言と言語設定ロジックを、独立したファイルに抽出し、各ページがそれを「インクルード（include）」できるようにします。
src/main/webapp/WEB-INF/commonに新しいファイル「i18n_setup.jspf」を作成し、先ほどのコードをそのままコピー＆ペーストします。
その後、他のJSPファイルに<%@ include file="/WEB-INF/common/i18n_setup.jspf" %>を追加すればOKです。
	5. その他解決すべき問題
現状では、ユーザーが選択した言語はセッション（session）に保存されています。これでは、ユーザーがウェブページを開くたびに、再度言語を選択し直す必要があるかもしれません。
今後は、それをデータベースに保存する必要があります。そうすれば、ユーザーがログインするたびに、デフォルトでそのユーザーが好む言語に切り替わるようになります。
